docker配置文件/etc/default/docker不生效

发表于 2017-12-04 | 更新于: 2017-12-04 | 分类于 docker

今天在修改docker daemon启动参数时发现修改/etc/default/docker文件不能生效，google之后总结了三种解决方法。

环境参数：

  • docker version : 17.05.0-ce
  • 系统版本 ubuntu 16.04.3 xenial

解决方法：

主要参考https://github.com/moby/moby/issues/9889

方法一：

编辑/lib/systemd/system/docker.service文件

方法二：

由于直接修改/lib/systemd/system/docker.service文件不太合适。

The file in /lib/systemd/system/docker.service is “reserved” for the package vendor.

相对优雅的做法是：

新增文件/etc/systemd/system/docker.service.d/docker.conf

同方法一，修改完成之后需要reload。

方法三：

docker官方提供了另外的方法来配置docker daemon, 可以通过新增/etc/docker/daemon.json文件来配置daemon启动项。https://docs.docker.com/engine/admin/

NOTE:需要注意的是，如果在daemon.json中配置了hosts参数，需要新增/etc/systemd/system/docker.service.d/docker.conf文件并配置，修改完成后reload即可生效。
详情查阅官方文档。
